# Story 1.1 — Slide Creator Web App: Setup Inicial e Interface Base

## Status

Done

## Story

**Como** usuário da plataforma EntrlacOS,
**quero** acessar uma interface web para criar apresentações de slides via IA,
**para que** eu possa criar decks profissionais sem precisar usar ferramentas externas.

## Contexto

O skill `slide-creator` existe em `.agents/skills/slide-creator/SKILL.md` como instruções para o modelo Claude. Esta story implementa a **camada web** que expõe esse skill via browser: um backend Node.js que carrega o SKILL.md como system prompt e chama a Anthropic API, servindo uma interface HTML/CSS/JS para o usuário.

Arquitetura definida pelo aiox-master:
- **Stack:** HTML/CSS/JS puro (frontend) + Node.js Express (backend)
- **Integração:** skill-loader.js lê SKILL.md em runtime → claude-client.js chama Anthropic API
- **Deploy:** Docker porta 3002 no servidor Contabo (144.91.115.235)
- **Localização:** `apps/slide-creator/`

## Acceptance Criteria

- [ ] AC1: Estrutura de pastas `apps/slide-creator/` criada conforme arquitetura definida
- [ ] AC2: `server.js` (Express) sobe na porta 3000 (mapeada para 3002 no Docker) sem erros
- [ ] AC3: `skill-loader.js` lê `SKILL.md` de `.agents/skills/slide-creator/` e retorna conteúdo como string
- [ ] AC4: `claude-client.js` envia briefing do usuário à Anthropic API com SKILL.md como system prompt e retorna resposta Markdown

- [ ] AC10: `ANTHROPIC_API_KEY` lida via variável de ambiente, nunca hardcoded

## Dev Notes

### Stack e Convenções

- **Runtime:** Node.js 20+ (LTS)
- **Framework:** Express 4.x
- **Frontend:** HTML/CSS/JS puro — sem bundler, sem framework (consistente com `apps/branding-os-showcase/`)
- **Anthropic SDK:** `@anthropic-ai/sdk` — instalar via `npm install @anthropic-ai/sdk`
- Código usa ES2022 com CommonJS (`require`/`module.exports`), 2-space indent, single quotes, semicolons

### Estrutura de Arquivos

```
apps/slide-creator/
├── Dockerfile
├── docker-compose.yml
├── package.json
├── .env.example
├── server.js                    # Entry point Express
├── src/
│   ├── skill-loader.js          # Lê SKILL.md do skill existente
│   ├── claude-client.js         # Chama Anthropic API
│   └── routes/
```

### Integração com Skill

`skill-loader.js` deve resolver o path do SKILL.md relativo ao root do projeto:

```javascript
// Path relativo ao root do monorepo
const SKILL_PATH = path.resolve(__dirname, '../../.agents/skills/slide-creator/SKILL.md');
```

O conteúdo do SKILL.md é passado como `system` prompt na chamada à API.

### Anthropic API

- Modelo: `claude-sonnet-4-6` (default — configurável via env `CLAUDE_MODEL`)
- `max_tokens`: 8192 (decks podem ser longos)
- Timeout: 120s (geração pode demorar)
- Streaming: não obrigatório nesta story (resposta completa OK)

### Docker e Deploy

- Porta interna: 3000
- Porta externa Contabo: 3002
- Variáveis de ambiente obrigatórias: `ANTHROPIC_API_KEY`, `PORT` (opcional, default 3000)
- `node_modules` no `.dockerignore`

### Dependências

- `express`: ^4.18.0
- `@anthropic-ai/sdk`: latest
- `marked`: ^9.0.0 (Markdown → HTML no frontend via CDN, não instalar no backend)
